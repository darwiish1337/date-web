# Will You? - Interactive Valentine Experience

An interactive web application designed to ask a special question with a playful, persistent "No" button that dodges the cursor.

[![GitHub](https://img.shields.io/badge/GitHub-View_on_GitHub-blue?style=for-the-badge&logo=github)](https://github.com/)

## Features

- **Interactive UI**: A modern, clean card-based design with smooth animations.
- **Smart Dodge Logic**: The "No" button intelligently avoids the cursor while staying within the card boundaries.
- **Responsive Design**: Optimized for both desktop and mobile devices.
- **Dynamic Overlays**: Fun visual feedback for both "Yes" and "No" (hover) interactions.
- **Clean Architecture**: Organized folder structure and OOP-based JavaScript implementation.

## Tech Stack

- **HTML5**: Semantic structure.
- **CSS3**: Custom properties, Flexbox, and advanced animations.
- **JavaScript (ES6+)**: Object-Oriented Programming (OOP) for clean, maintainable logic.

## Project Structure

```text
├── assets/          # GIF assets and images
├── css/             # Stylesheets
├── js/              # Application logic (OOP)
├── index.html       # Main entry point
└── README.md        # Project documentation
```

## Implementation Details

- **OOP Design**: The application logic is encapsulated within the `ValentineApp` class, ensuring separation of concerns.
- **SOLID Principles**: Focused on Single Responsibility for functions like `getRandomPos` and `moveNoBtn`.
- **Best Practices**: Event delegation, DOM caching, and clean commenting in English.

## How to Run

Simply open `index.html` in any modern web browser.
